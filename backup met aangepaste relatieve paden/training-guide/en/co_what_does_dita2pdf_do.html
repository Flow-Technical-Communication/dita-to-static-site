What does DITA2PDF do?

Simply put, DITA2PDF follows these steps:

DITA-FMx

Once you have written your documentation in Oxygen, you can import it into Adobe FrameMaker. You do this with the DITA-FMx plugin that you can buy and download from http://leximation.com/dita-fmx/. This plugin allows you to create and edit DITA files in FrameMaker and provides many enhancements and features beyond those available with FrameMaker's standard DITA functionality.

Essentially, DITA-FMx is similar to a transformation scenario but it is located within FrameMaker instead of Oxygen. It works by generating a FrameMaker bookfile from your DITA (book)map. It does not generate FM files for every topic you have, but groups all of your nested topics in one file per stub topic/topic head.

The lay-out of your bookfile is specified according to a default ditafmx-bookbuild.ini file. This default file is automatically created by the plugin. In order to make it use your templates instead, you can customize this default file and direct it to use the FrameMaker templates you have developed for your documentation.

Tip: As you cannot adjust the filename of the bookbuild file, you instead save these customized files in separate folders. This way, you can create a separate template for every type of documentation.

FrameMaker

Once DITA-FMx has generated a bookfile in FrameMaker, you get to see what your documentation will look like when you publish it to PDF. While you can adjust this lay-out straight away in FrameMaker, DITA2PDF allows for easier, global lay-out strategies. This means you can change lay-out once and ensure the plugin generates a bookfile in which those changes are automatically applied every time:
  • Template: Any changes you make that should happen on a global level can usually be specified in the templates. When you generate your file, the template automatically applies the correct type of lay-out everywhere. For example: adjust colours, fonts, line spacing ...
  • Oxygen: For more specific changes you can often make adjustments in the original DITA files in Oxygen. You can adjust the files in FrameMaker and then you can enter the adjusted values into Oxygen so they will be used each time you generate your bookfile. For instance, you can adjust the dimensions of your table, or set the dpi of your images in an attribute, directly in Oxygen.

When you are happy with the lay-out of your documentation, save the bookfile as PDF.
