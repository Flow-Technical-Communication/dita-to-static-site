Guidelines for using DITA2PDF

Specify a folder per template

If your documentation will be used for different outputs and templates, it can be easy to lose track of all of your files. As the ditafmx-bookbuild.ini file always needs to have the same name, you cannot add product names to it in order to distinguish between variants. Therefore it is recommended you make a pdf_[product name] folder per product. In this folder you then add:
  • [product].ditaval: A unique DITAVAL file suited to filter and flag the information for this product.
  • PDF_templates: A folder with the Adobe FrameMaker templates you have created for this product.
  • ditafmx-bookbuild.ini: A unique bookbuild file which specifies the requirements for this product.

Applying a template

In order to apply a template when a bookfile is generated, you create unique FrameMaker template files which you store in their own folder. Next, add a string to your ditafmx-bookbuild.ini file under [General] in which you specify which folder contains the templates. The correct template will be applied when the bookfile is generated.

For example, you created your own FrameMaker templates and saved them in a folder named "PDF_templates". In your ditafmx-bookbuild.ini file, under [General] you add BookTemplatesDir=PDF_templates. The DITA-FMx will use the templates in that folder when it generates your bookfile.

Applying a DITAVAL

You can customize the ditafmx-bookbuild.ini file to apply the correct template, but you can also use it to apply DITAVAL filtering. Instead of specifying a DITAVAL in your transformation scenario, specify it in your ditafmx-bookbuild.ini file: add two strings under [BookBuildOverrides].

For example, you want to filter Word 2003 references from your generated FrameMaker bookfile. You create the appropriate DITAVAL file and add it to your PDF output folder which also contains the folder with your templates. In your ditafmx-bookbuild.ini, you add under [BookBuildOverrides]:
ApplyDitaval=1
DitavalName=./examples.ditaval

Resizing tables

The display of your tables in Oxygen will rarely match what you want in the output. To ensure tables look the same each time you generate a bookfile from your DITA files: resize your tables manually in FrameMaker and then adjust them in Oxygen. Then you select and right-click a column and choose Resize columns. Copy the value you find next to To Width and paste it into your corresponding Oxygen topic in the <colspecs> element of the corresponding column.

CAUTION: It is advised to use natural numbers (for example 1, 2, 3...) rather than numbers containing a decimal point to avoid transformation errors.
