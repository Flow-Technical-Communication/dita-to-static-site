Ditamap attributes

DITA maps enable authors to create structures specific to the output format, using specific attributes. These attributes are therefore not available on single topics, as these should be reusable, regardless of the output format. The following attributes and corresponding values are specific for ditamaps:

Attribute Function
@collection-type See What does collection-type do?. This attribute specifies how the children of a <topicref> element relate to their parent (which holds the attribute) and to each other. It determines how navigation links are generated in topics. Possible values are:
  • Unordered: Links are generated from parent to children and vice versa
  • Family: Links are generated from parent to children, vice versa and from sibling to sibling
  • Sequence: Links are generated from parent to children, vice versa from child to previous sibling and next sibling
  • Choice: Reader needs to select one child topic to proceed
@linking See What does @linking do?. This attribute enables an author to specify how a topic should participate in a relationship. The following values are valid:
  • None: The topic cannot be linked to or link to other topics.
  • Sourceonly: The topic will link to its related topics but not vice versa.
  • Targetonly: The related topics will link to it but not vice versa.
  • Normal: The topic will link to related topics and vice versa.
Note: Authors also can create links directly in a topic by using the <xref> or <link> elements, but in most cases map-based linking is preferable, because links in topics create dependencies between topics that can hinder reuse.
@processing-role This attribute specifies whether the topic or map referenced is:
  • Normal: The topic is a readable part.
  • Resource-only: The topic is merely a resource in order to resolve key or content references.
@chunk This attribute provides the following output results:
  • Multi-topic files transform into smaller files.
  • Individual DITA topics combine into a single file.
Tip: If you add a glossary to a webhelp, you might want to use the @chunk attribute. If not, each glossary term will have its own web page.
@copy-to This attribute specifies whether a duplicate version of the topic is created when it is transformed. The value specifies the URI to be referenced by a @conref attribute, <topicref> element or <xref> element.
@print Specifies whether the topic is included in printed output.
@search Specifies whether the topic is included in search indexes.
@toc Specifies the location where a table of contents should be generated.
@navtitle Deprecated. Use the <navtitle> element instead.
@locktitle Deprecated. Use the <navtitle> element instead.
